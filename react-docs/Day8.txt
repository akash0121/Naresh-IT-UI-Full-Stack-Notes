Components in React Vite App:
	- It is mandatory to keep all components in “src” folder.
	- Every component can have 3 files 
			a) .jsx [mandatory]
			b) .css
			c) .spec.js  (or) .test.js 
	- JSX file comprises of presentation & logic.
	- CSS file comprises of styles.
	- spec / test files comprises of test methods for testing component functions & UI.

			login.jsx
			login.css
			login.spec.js / login.test.js

	 - Every component can be a function or class. 
	 - Component name must start with an uppercase letter.
	 - Component must return a JSX element as one fragment. 
	 - JSX fragment can be any one of the following 

				a) A semantic or non-semantic element of HTML
							<header> <div> <main> <nav> etc..

				b) An empty fragment without dynamic features

							< >
								… your content 
							</>

				c) React.Fragment

							<React.Fragment>
								… your content …
							</React.Fragment>

Syntax:
				export function Register()
				{
				    return (
							   <>
								  <h2>Register</h2>
							   </>
							  )
				}
							  (or)
	
				function Register() 
				{
					return ( <> </> )
				}
				export default Register;

							  (or)

				const Register = ()=> ( <> </> );
				export default Register;

	- JSX element must have end token.
	- JSX element will not allow attributes. You have to use properties. 
	- Component styles are defined in a CSS file or configured inline. 
	- Make sure that styles are not defined using “type selector”. 
	  [ use type selector if you want element same across all components ]
	- CSS files must be imported into component.
		
					import   “./register.css”;

Enable Bootstrap for React vite App:
	 - Install bootstrap for project

			> npm install  bootstrap  bootstrap-icons --save

	- Import bootstrap files into “main.jsx”

		   import  ‘../node_modules/bootstrap/dist/css/bootstrap.css’ ;
		   import  ‘../node_modules/bootstrap-icons/font/bootstrap-icons.css’;

Note: You can install the extension for VS code 
		“intelliSense for CSS class names in HTML”.

	 - You can directly uses bootstrap and icon classes for elements

			<li className=“bi bi-facebook”> </li>
			<input type=“text” className=“form-control” />

												 React Data Binding
 - Data binding is a technique of accessing data from source and bind to UI elements. It identifies the changes in UI and updates into source. 
 - Data binding techniques are classified into 2 types
		1. One Way Binding
		2. Two Way Binding
 - React implicitly supports One Way Binding. 
 - It makes the data in UI elements read-only. 
 - You can bind data using a data binding expression “{ }”.

Syntax:
			var user = “John”;

			<p> Hello ! { user } </p>

			<input type=“text”  value={user} />
