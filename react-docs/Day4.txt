React Components:
	- A component is a reusable design with styles and logic. 
	- Technically component comprises of 
			a) Presentation 
			b) Styles
			c) Functionality 
	- Presentation with HTM
	- Styles with CSS 
	- Functionality with JSX or TypeScript [TSX]
	- Component based design allows
			a) Reusability 
			b) Extensibility 
			c) Separation 
			d) Maintainability 
			e) Testability 

Designing a Component:
	 - A component is designed by using JavaScript “function” or “Class”.
	 - Class components are still supported in React but not recommended for modern code.

Function Component:
	 1. You can configure a function by using 2 techniques
				a) Declaration
				b) Expression 

		Syntax: Declaration
					function Login() 
					{
					}

		Syntax: Expression
				  const Login = function() {
				  }
	
		FAQ: What is difference between declaration and expression?
		Ans: Expression will not allow hoisting. 
			    Declaration allows hoisting. 

	2. Function component must return markup, It can’t be void type. 
		
		- Component must return JSX element. [JavaScript XML]

		Syntax:
					function Login()
					{
					   return ( <JSXElement> </JSXElement> );
					}

		- Function component must return only “One Fragment”.
			
					function Login()
					{
						return(
								<h1> Login </h1>        => Invalid : JSX must have only one parent
								<form> </form>
								)
					}

					function Login()
					{
						return (
								<form>
									<h1> Login </h1>      => valid
								</form>
								)
					}

		- Every JSX element must have “end token”.
		
					<img>					// invalid
					<img> </img> 		// valid					<img />					// valid  - self ending 

		- JSX element can’t have “attributes”, It must use only “property”.

					<img   class=“img-fluid” />        // invalid : class is an attribute 

					<img   className=“img-fluid” />   // valid : className is a property 

		FAQ: What is difference between an attribute & property?
		Ans:    Attribute is immutable & property is mutable. 

		- A component can be accessed as “Token”. 

					<Login />
