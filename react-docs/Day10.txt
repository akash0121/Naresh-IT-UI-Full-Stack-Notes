Boolean Type:
		- Boolean type handles “true” & “false”.
		- React can use boolean value but can’t display the boolean value. 
		- You have to render conditional data using boolean. 
		
Note: JSX will not allow any statements inside JSX element. 
		 
		- You have to handle all actions on boolean by using operators or methods. 

Syntax:
			const [stock, setStock] = useState(true);

			<p> { (stock===true) ? “true” : “false” } </p>

Undefined Type
		- It is same as in JS.
		- It is configured and verified using “undefined” keyword. 
		- React can’t display “undefined” keyword. It can use undefined. 

Syntax:
			const [ price, setPrice ] = useState();

			<p>  { (price) ? price : “Price not defined”  } </p>

Null Type
		- It is used to define values at runtime. 
		- If a value is not provided into reference at runtime then it uses “null”. 
		- React can’t display null, but can use null as keyword.

Syntax:
			const [ price, setPrice ] = useState();

			setPrice(prompt(“Enter Price”));

			<p> { (price===null) ? “Price Required”: price } </p>

Symbol Type 
		- It is same as in JS.	
		- It is used to hide a key in object. 

Syntax:
			var Id = Symbol();

			var product = { 
				[Id] : 1,
				Name: “TV”
			 }

			product[Id];        // to access individually. 

Array Type
		- All array rules same as in JS.
		- Array configuration.
		- Array Elements.
		- Array Methods.
		- Array de-structure is same as in JS.
		- Array spread approach is same. 
Ex: data-binding.jsx

import { useState, useEffect } from "react";

export function DataBinding(){
    
    const [electronics] = useState(["TV", "Mobile"]);
    const [fashion] = useState(["Kids Fashion", "Women Fashion"]);
    const [categories, setCategories] = useState([]);
   
    useEffect(()=>{

        setCategories([...electronics, ...fashion]);
        console.log(categories);
        
    },[]);


    return(
        <div className="container-fluid p-2">
            <p> {categories} </p>
        </div>
    )
}


	- Array presentation in React requires “map()” method as iterator to read all elements and generate UI element for items list. 
	- Every child item in list must have a unique “key” reference. 

Syntax:
			<ul>
				{
				   collection.map( (value, index) => <li key={index}> { value } </li> )
				}
			</ul>
